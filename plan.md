# 多模态大模型学习代码 - 开发笔记

## 项目目标

创建一个用于学习的多模态大模型代码，具有以下特点：
- 最简单的结构，最少的层数
- 方便debug查看中间变量
- 随机初始化权重，不需要下载预训练模型
- 目标是理解多模态模型的框架和处理流程

## 开发思路

### 整体架构设计

采用类似LLaVA的基本结构，但大幅简化：

```
输入图像 -> Vision Encoder -> 图像特征
                                    |
                                    v
                              Cross Attention Fusion
                                    ^
                                    |
输入文字 -> Text Encoder -> 文字特征

融合特征 -> Text Decoder -> 输出文字
```

### 技术选型

#### 1. 位置编码方案
- **文本**: 使用RoPE（旋转位置编码）
  - 原因: 现代LLM的标准做法，相对位置编码，适合长序列
  - 优势: 比传统绝对位置编码更好地捕捉相对位置关系

- **图像**: 使用2D位置编码
  - 原因: 图像是2D结构，需要编码x和y两个维度
  - 实现: 可学习的位置embedding或固定的sinusoidal编码

#### 2. 标准化策略
- 使用Pre-LayerNorm（现代Transformer标准）
- 在每个子层之前应用LayerNorm
- 优势: 训练更稳定，收敛更快

#### 3. 残差连接
- 在每个Transformer块中使用
- 帮助梯度流动，避免梯度消失

### 模块设计

#### Vision Encoder（图像编码器）
1. **Patch Embedding**
   - 使用卷积层实现（kernel_size=patch_size, stride=patch_size）
   - 将图像分成16个patches (4x4网格，每个patch为16x16)

2. **2D Position Encoding**
   - 为每个patch添加位置信息
   - 使用可学习的位置编码（更灵活）

3. **Transformer处理**
   - 1层Transformer提取视觉特征
   - Multi-Head Attention + FFN

4. **输出**: [batch, 16, 128] - 16个视觉token

#### Text Encoder（文本编码器）
1. **Token Embedding**
   - 简单的nn.Embedding层
   - vocab_size=1000，适合教学

2. **RoPE位置编码**
   - 在attention计算时应用到Q和K
   - 通过旋转变换注入位置信息

3. **Transformer处理**
   - 1层Transformer提取文本特征
   - 使用带RoPE的自定义TransformerBlock

4. **输出**: [batch, seq_len, 128]

#### Fusion Layer（跨模态融合层）
1. **Cross Attention**
   - Query: 文本特征（"我想从图像中查询什么？"）
   - Key & Value: 图像特征（提供视觉信息）
   - 这样文本可以"看到"图像中相关的内容

2. **Feed-Forward**
   - 进一步处理融合后的特征

3. **输出**: [batch, seq_len, 128] - 融合后的特征

#### Text Decoder（文本解码器）
1. **Decoder Transformer**
   - 1层Transformer处理融合特征
   - 使用Causal Mask确保自回归特性
   - 带RoPE位置编码

2. **Language Model Head**
   - Linear层投影到词汇表: [batch, seq_len, vocab_size]

3. **输出**: logits用于预测下一个token

### 超参数选择

```python
hidden_dim = 128          # 足够小，快速训练，但仍有表达能力
num_heads = 4             # 保证head_dim=32，合理的大小
num_layers = 1            # 最简单，容易理解
vocab_size = 1000         # 小词汇表，适合教学
image_size = 64           # 小图像，快速处理
patch_size = 16           # 得到4x4=16个patches
max_seq_len = 32          # 短序列，快速测试
```

这些参数的选择平衡了：
- **计算效率**: 可在CPU上快速运行
- **理解性**: 参数量少，容易观察
- **功能性**: 足以演示多模态模型的核心机制

### Debug功能设计

#### 前向传播调试
在每个关键步骤打印：
- 输入/输出的shape
- 统计信息（均值、标准差、最大最小值）
- 异常值检查（NaN、Inf）

#### 反向传播调试
打印每层的梯度信息：
- 梯度均值、标准差
- 梯度范数（检测梯度消失/爆炸）
- 前5层和后5层的对比

#### Attention可视化
- 打印attention权重的top-k值
- 显示哪些位置的attention最高
- 帮助理解模型在"看"什么

### 实现细节与坑

#### RoPE实现
**关键点**:
- 需要在attention计算前应用到Q和K
- 频率随维度递减: 1/(base^(2i/dim))
- 需要处理head维度: [batch, seq_len, num_heads, head_dim]

**容易出错的地方**:
- 维度转换顺序要正确
- sin和cos需要正确广播
- _rotate_half函数的实现要精确

#### Cross Attention实现
**关键点**:
- Query来自文本，Key/Value来自图像
- 这是单向的信息流动（文本查询图像）
- 与Self-Attention的区别在于输入来源

#### Causal Mask
**为什么需要**:
- 文本生成是自回归的
- 位置i不能看到位置>i的信息
- 使用下三角矩阵实现

#### 权重初始化
**策略**:
- Linear: xavier_uniform
- Embedding: normal(0, 0.02)
- LayerNorm: weight=1, bias=0

**原因**:
- 避免梯度消失/爆炸
- 确保训练初期的稳定性

### 后续扩展方向

#### 重要实现说明

**自回归训练中的Labels对齐**

在自回归语言模型训练中，需要对齐logits和labels，确保：
- 位置i的输入预测位置i+1的标签
- 这是标准的"预测下一个token"训练方式

**实现位置**: `models/multimodal_model.py:218-219`

**核心代码**:
```python
# 对齐logits和labels
shift_logits = logits[:, :-1, :].contiguous()  # 取位置0到N-1的预测
shift_labels = labels[:, 1:].contiguous()      # 取位置1到N的标签
```

**为什么需要对齐**:
1. 自回归的本质是预测下一个token
2. 防止训练时的信息泄露（看到正确答案）
3. 确保训练和推理的一致性

详细原理说明见: `自回归训练原理说明.md`

### 开发过程中的思考

#### 为什么选择这个架构？
- **简单性**: 单向融合（文本查询图像），最容易理解
- **经典性**: 类似LLaVA，业界验证过的方案
- **扩展性**: 可以逐步增加复杂度

#### 为什么用RoPE而不是传统位置编码？
- RoPE是现代LLM的标准
- 学习RoPE对理解新模型很重要
- 相对位置编码的优势明显

#### 为什么这么小的模型？
- **学习优先**: 专注理解原理
- **快速迭代**: CPU上快速运行
- **观察方便**: 参数少，容易调试

### 常见问题与解决方案

#### Q: Loss不下降怎么办？
**分析**:
- 随机数据本身没有模式
- 这是正常的，不影响学习流程

**如果用真实数据**:
- 检查学习率（尝试1e-3到1e-5）
- 检查梯度（是否消失或爆炸）
- 检查数据（标签是否正确）

#### Q: 出现NaN怎么办？
**常见原因**:
- 学习率太大
- 梯度爆炸
- 数值不稳定

**解决方法**:
- 降低学习率
- 使用梯度裁剪
- 检查权重初始化

#### Q: 显存不够怎么办？
**调整**:
- 减小batch_size
- 减小hidden_dim
- 减小图像大小
